# 问题修复说明

## 问题描述

用户反馈了两个问题：
1. 退出登录时提示"登录失败！用户名或密码错误"
2. 点击入门模块的"开始演示"会弹窗

## 问题原因

### 问题1：退出登录错误提示

**原因：**
- `csrf/logout.php` 重定向到 `login.php`
- `login.php` 是一个登录处理脚本，不是登录页面
- 直接访问 `login.php` 时，因为没有POST数据，会执行错误处理代码
- 错误处理代码使用了 JavaScript alert 弹窗：`alert("登录失败！用户名或密码错误")`

**修复：**
1. 修改 `csrf/logout.php`，重定向到 `index.php` 而不是 `login.php`
2. 修改 `login.php`，添加请求方法检查，非POST请求直接重定向到 `index.php`

### 问题2：开始演示弹窗

**原因：**
- 用户未登录时点击"开始演示"按钮
- 被重定向回 `index.php`（这是正常的登录检查）
- 如果此时触发了 `login.php`，就会显示错误弹窗

**修复：**
- 通过修复问题1，问题2也随之解决
- 现在未登录用户点击"开始演示"会被重定向到首页登录框，不会弹窗

## 已修复的文件

### 1. `csrf/logout.php`
```php
// 修改前
header('Location: login.php');

// 修改后
header('Location: index.php');
```

### 2. `csrf/login.php`
```php
// 添加请求方法检查
if ($_SERVER['REQUEST_METHOD'] !== 'POST') {
    header('Location: index.php');
    exit;
}
```
