💻 攻击代码与防护对比

// 攻击代码（失败）
<!-- 攻击者尝试预测Token -->
<form action="csrf_token_weak.php" method="POST">
  <input type="hidden" name="action" value="update_strong">
  <input type="hidden" name="nickname" value="被攻击的用户">
  <!-- ❌ 攻击者只能伪造短Token -->
  <input type="hidden" name="csrf_token" value="fake12345">
</form>

// 防护代码（成功）
// 强Token生成
$token = bin2hex(random_bytes(32)); // 64位

// Token验证
if (strlen($token) !== 64) {
  die('Token长度错误');
}
if ($token !== $_SESSION['strong_token']) {
  die('Token验证失败');
}

// 为什么攻击失败？
1. 强Token长度为64位，攻击者无法预测
2. 攻击者只能伪造8位短Token
3. 服务器验证Token长度失败（8 ≠ 64）
4. 请求被拒绝，用户信息保持安全

💡 强Token vs 弱Token

弱Token（可预测）：基于时间戳生成，长度短（8位），攻击者可以预测并构造有效Token。
强Token（不可预测）：使用加密安全随机数生成器，长度足够（64位），攻击者无法预测，防护有效。
