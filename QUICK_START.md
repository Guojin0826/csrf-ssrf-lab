## 教学建议

### CSRF 教学流程

1. **讲解原理**（5分钟）
   - CSRF 是什么？
   - 攻击者如何构造恶意页面？
   - 为什么用户会受到影响？

2. **演示攻击**（10分钟）
   - 先演示银行转账场景
   - 再演示论坛密码修改场景
   - 让学生亲自体验攻击过程

3. **讲解防护**（5分钟）
   - CSRF Token 的作用
   - Referer 验证
   - SameSite Cookie

4. **对比演示**（5分钟）
   - 展示安全版本的防护效果
   - 让学生理解防护措施的重要性

### SSRF 教学流程

1. **讲解原理**（5分钟）
   - SSRF 是什么？
   - 为什么服务器会发起请求？
   - 攻击者能做什么？

   - 演示访问内网服务

## 常见问题

**Q: 为什么攻击页面能成功？**
A: 因为漏洞版本没有 CSRF Token 保护，浏览器会自动携带用户的登录凭证。

**Q: 安全版本为什么能防护？**
A: 安全版本包含 CSRF Token，攻击页面无法获取正确的 Token，请求会被拒绝。

**Q: SSRF 能做什么？**
A: 可以访问内网服务、读取本地文件、探测端口、攻击内部系统等。

**Q: 如何彻底防护 SSRF？**
A: 使用严格的 URL 白名单、禁止内网访问、禁用危险协议、网络隔离等。
